1. Project Overview

A vertical marketplace platform connecting fighters, coaches, gyms, and vendors. Users

can buy gear, book coaches, discover gyms, access premium training content, attend

VVIP events, and communicate via E2EE private chat.

7. Phase 5 — E2EE Chat

2 days Private encrypted messaging

7.1 Architecture

User A                          Server                      User B

   │                              │                           │

   ├─ Generate RSA-OAEP key pair ─┤                           │

   │                              │                           │

   ├─ POST /api/chat/keys/ ──────►│  store public_key         │

   │                              │                           │

   ├─ GET /api/chat/keys/B ──────►│                           │

   │◄── B's public_key ──────────┤                           │

   │                              │                           │

   ├─ Derive shared AES-GCM key   │                           │

   │                              │                           │
   ├─ Encrypt msg ───────────────►│  relay ciphertext ───────►│

   │  ciphertext + nonce          │                           ├─ Decrypt

   │                              │                           │  → plaintext

7.2 Models

• 

ChatConversation: participant1, participant2, created_at, last_message_at

• 

ChatMessage: conversation, sender, encrypted_body, nonce, created_at, read

• 

UserKey: user, public_key (JWK), created_at, rotated_at

7.3 WebSocket

WS /ws/chat/<conversation_id>/?token=<jwt>

Consumer validates JWT on connect, adds user to conversation group.

7.4 Client-Side Crypto (Web Crypto API)

• 

generateKeyPair() → RSA-OAEP 2048-bit

• 

encrypt(plaintext, key) → { ciphertext, nonce }

• 

decrypt(ciphertext, nonce, key) → plaintext

• 

Private key stored in IndexedDB (never leaves browser)
